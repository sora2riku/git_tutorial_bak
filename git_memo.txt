==================
変更差分を確認する
==================
#git addする前の変更分
git diff
git diff <ファイル名>

#git addした後の変更分
git diff --staged

git diff
+がついているところが変更箇所

qをおして抜ける

==================
現在の変更状況を確認する。
==================

git status
下記の情報が表される
ワークツリーとステージの間で変更されたファイル
ステージとリポジトリの間で変更されたファイル

変更されたファイルをステージに追加する
git add index.html

コミットする
git commit

git statusで変更状況を確認する


==================
変更を記録する（コミット）
==================
ステージの内容をリポジトリにスナップショットを記録する。
ツリーファイルとコミットファイルが作られる
git commit
git commit -m "<メッセージ>"
git commit -v


git commitを実行するとGitEditerが起動する
メッセージを入力する

保存して終了
:write out
:exit

保存しないで終了
:qa!
:exit

メッセージ例（個人開発時）
1行目：変更の内容と要点を偉業で簡潔に書く

メッセージ例（チーム開発時）
1行目：変更の要約
2行目：空行
3行目：変更した理由

==================
GITHUBのメモ
==================


ターミナルの頻出コマンドについてまとめます。
Gitはターミナルで作業するので、その時に絶対に覚えておいてほしいコマンドだけまとめます。

cdコマンド

ディレクトリを移動します。

lsコマンド

ディレクトリの内容を表示します。 ls -a コマンドで、隠しファイルを含めたディレクトリ全内容を表示します。

mkdirコマンド

ディレクトリを新規作成します。

rmコマンド

ファイルを削除します。

cpコマンド

ファイルをコピーします。

mvコマンド

ファイルの移動とファイル名の変更を行います。

catコマンド

ファイルの中身を表示します。
